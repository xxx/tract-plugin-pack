# HD26 — Hyper/Dimension Recreation — Design Spec

- **Date:** 2026-05-30
- **Status:** Design approved; pending written-spec review → implementation plan
- **Crate:** `hd26` (display name: **HD26**)
- **Type:** nih-plug audio effect (VST3 / CLAP / standalone), CPU-rendered GUI

## 1. Goal

Recreate the **Hyper/Dimension** FX module from Xfer Records Serum as a standalone insert effect, accurately, as a **superset**: faithful Serum behavior is the default, with a small set of clearly-separated extras. Two independent stages in Serum's order — **Hyper** (a multi-voice modulated-delay unison/chorus) then **Dimension** (a pseudo-stereo widener) — each with its own Mix.

This is an *accurate recreation* project: the DSP below is grounded in the Serum manual plus DSP literature (see §13), not guesswork.

## 2. Key research findings that shape the design

1. **Hyper is a "micro-delay chorus" of 1–7 voices** whose pitch *oscillates* sharp/flat (manual, verbatim). The correct, lowest-CPU, zero-latency implementation is a **multi-voice modulated fractional-delay line** (one shared ring per channel, N read taps, each delay swept by its own LFO; Doppler gives the detune: `pitch ratio = 1 − dD/dt`). Confirmed by Julius O. Smith / CCRMA.
2. **Serum's Dimension modulates AMPLITUDE, not pitch/delay.** The manual states Dimension "acts in a similar way to Hyper, but instead of detuning the voices, it delays them and modifies their amplitude," and is "4 delay lines summed out-of-phase and slowly amplitude modulated." This is the **opposite** of the real Roland Dimension-D (which modulates delay time). An accurate Serum clone must use **amplitude modulation** — a 4-tap Lauridsen-style complementary-comb widener summed antisymmetrically so it **collapses cleanly to mono**. A literal Dimension-D (pitch-modulated) implementation is offered as a *non-default* mode.
3. **Voice spread:** the JP-8000 supersaw distributes detune **asymmetrically and nonlinearly** (Adam Szabo, measured) — subtle until ~halfway, dramatic near max. We adopt this shape for the Hyper voice spread.
4. **Detune vs. Rate are coupled in raw delay modulation** (`peak cents ∝ rate × depth`). Scaling depth ∝ 1/rate (with a clamp) decouples them so Detune sets the cents of spread and Rate only sets speed.
5. **Retrig needs a trigger source.** As an audio insert we have no guaranteed MIDI, so Retrig is driven by an **audio transient detector** (fast-vs-slow envelope), not note-ons.

## 3. Scope

### In scope
- Hyper stage: Unison, Detune, Rate, Width (extra), Retrig + Sensitivity (transient-driven), Mix.
- Dimension stage: Size, Mode (AM "Serum" / Pitch "Dimension-D"), Wet HPF (extra), Mix.
- Global: Output gain, Bypass.
- Minimal knob-panel GUI + a focused **retrig activity indicator** (LED + input-level bar).
- Zero reported latency; no allocations on the audio thread; lightweight enough for many instances.

### Out of scope (deliberately deferred)
- Tempo-sync of Rate (free Hz only — Serum behavior).
- MIDI note-on retrig (transient-only).
- Full visualization (goniometer / voice-spread display).
- Promotion of the delay-line primitive to `tract-dsp` (kept local until a second consumer exists; YAGNI).

## 4. Signal flow

```
in(L,R) ──► [ HYPER ] ──► [ DIMENSION ] ──► [ Output gain ] ──► out(L,R)
                 │              │
           own Mix knob   own Mix knob (0% = stage disabled)
```

Each stage is internally parallel dry/wet (its Mix). Stages are independent: Unison=0 bypasses Hyper; Dimension Mix=0 bypasses Dimension. Both feed-forward → **0 reported latency**.

## 5. Hyper stage (`hyper.rs`)

A multi-voice modulated-delay chorus, run **per input channel** (preserves input stereo) over a shared per-channel ring buffer.

- **Voices:** Unison ∈ 0–7. 0 = stage bypass (dry). Each voice = one interpolated read tap at delay `Dᵢ(t) = D₀ + detune_offsetᵢ-derived sweep`, summed (with per-voice gain).
- **Voice spread:** voice detune positions follow a **normalized asymmetric spread** matching Szabo's measured JP-8000 distribution at N=7 (`{−0.110, −0.063, −0.020, 0, +0.020, +0.062, +0.107}`), sampled for the active N (center voice present for odd N). Positions are scaled by the **Detune** knob through Szabo's nonlinear response curve.
- **Detune → cents:** Detune maps to a peak pitch deviation per voice (target max on the order of **±50 cents**, finalized during voicing). Realized via delay-modulation depth using `cents_peak ≈ 1731 · 2π · rate · depth`.
- **Detune/Rate decoupling:** `depth = clamp( target_cents / (1731 · 2π · rate), 0, MAX_DEPTH )` so Detune governs cents and Rate governs speed. `MAX_DEPTH` bounds the buffer (see RT-safety).
- **Rate:** 0.01–10 Hz free-running LFO (per-voice phase staggered `2π·i/N`, with small incommensurate rate spread to avoid coherent vibrato). Phase tracked in *turns* `[0,1)` (sample-rate-independent), per the `multosis` tremolo pattern.
- **Interpolation:** **4-point cubic (Lagrange/Hermite)** per tap — clean highs under modulation, magnitude ≤ unity.
- **Width (extra):** 0 = L/R voice banks use identical LFO phases → coherent → mono-safe thickening that preserves the input's own stereo. >0 introduces inter-channel LFO-phase offset + voice panning → decorrelation/widening.
- **Retrig:** when on, a fired transient (§6) **zeros all voice LFO phases** → voices fan out from center (the "laser zap").
- **Mix:** linear dry/wet blend of the thickened signal.

### RT-safety
- Per-channel ring sized once at `initialize()`/`reset()` for `D₀ + MAX_DEPTH` at the max sample rate (target max ring ≈ 50 ms → ~9.6k samples @ 192 kHz). Never reallocated in `process()`.
- Unison count changes are **gain-ramped** (fade voices in/out) to avoid clicks.
- All params smoothed; LFO/delay trajectories are inherently continuous.

## 6. Transient detector (`transient.rs`)

Drives Retrig. Audio-thread-safe, allocation-free.

- **Fast envelope** (short attack ~1–5 ms, release ~20–50 ms) vs. **slow envelope** (~50–150 ms).
- **Detection function:** fast-vs-slow gap/ratio (input-gain robust). **Fire** on a rising edge crossing the **Sensitivity**-derived threshold, then enforce a **refractory hold** (~20–50 ms) before the next fire.
- On fire: signal Hyper to zero voice phases, and bump the GUI trigger counter (§9).
- **Sensitivity** maps to the threshold (higher sensitivity → lower threshold). Only active when Retrig is on.

## 7. Dimension stage (`dimension.rs`)

Shared **4-delay-line** structure; **Size** scales base delay `τ_base` (~1–20 ms, exponential) with the four taps at fixed ratios (e.g. `{0.6, 0.85, 1.0, 1.4}`) so the comb pattern shifts coherently. **Mode** selects what is modulated:

### AM mode — "Serum" (default, accurate)
- Taps fed from the **mono sum**; each tap multiplied by a **slow internal amplitude LFO** (~0.15–0.6 Hz, depth ~0.2, phases staggered 0/90/180/270°).
- Summed **out-of-phase into a pure side signal**: `+wet → L`, `−wet → R`.
- ⇒ `L_out + R_out` cancels the wet → **collapses to mono** (mono-safe; expect level drop toward silence at 100% wet in mono, as documented).
- This is a 4-tap **Lauridsen complementary-comb** widener animated by tap-gain tremolo.

### Pitch mode — "Dimension-D" (non-default flavor)
- Delay **times** modulated by slow **anti-phase** LFOs (~0.25–0.5 Hz, ±1.5–2.5 ms around ~7.5–10 ms base), with **opposite-polarity L↔R cross-feed** — the classic BBD vibrato-chorus character.
- Reduced mono compatibility (documented cross-feed tradeoff). Not a literal SDD-320 clone (the real unit has 2 lines); a 4-line flavor sharing this stage's structure.

### Common
- **Wet HPF (extra):** high-pass on the side/wet signal (20–500 Hz, default ~120 Hz) so low end stays mono/solid. Biquad/1-pole → no added latency.
- **Mix:** 0% disables the stage; else blends wet in.

## 9. GUI (`editor.rs`)

softbuffer + tiny-skia + fontdue, freely resizable (scale = physical_width / WINDOW_WIDTH), size persisted via `EditorState`. Shared widgets: `param_dial`, stepped-selector, toggles. Two labeled sections + global row + retrig indicator:

```
+------------------- HYPER -------------------+------- DIMENSION -------+
|  (Unison) (Detune) (Rate) (Width)           |  (Size)   (Wet HPF)     |
|  [Retrig] (Sens)            (Mix)           |  [Mode: AM | Pitch]     |
|  ┌ retrig activity ───────────────┐         |          (Mix)          |
|  │ [||||||||      ] level   (O)LED │         |                         |
|  └─────────────────────────────────┘         |                         |
+---------------------------------------------+-------------------------+
                     (Output)     [ Bypass ]
```

### Retrig activity indicator (LED + level bar)
- **Input level bar:** the detector's fast envelope.
- **Trigger LED:** flashes on each *fired* retrig (reflects the refractory hold, so it shows real retrigs, not raw threshold crossings); fades over a few frames.
- User tunes by watching the LED while turning Sensitivity.

### Audio → GUI telemetry (lock-free, no alloc)
Shared `Arc` struct read each editor frame, written by the audio thread (same pattern as `imagine` vectorscope / six-pack `band_activity`):
- `level: AtomicU32` — fast-envelope value (f32 bit pattern).
- `trigger_count: AtomicU32` — incremented per fired retrig; editor compares to last-seen to detect new fires and drive the LED flash.

(Right-click-to-type text entry applies to continuous dials per the shared `TextEditState`; toggles/stepped selectors are non-editable.)

## 10. Module / file structure

```
hd26/
  Cargo.toml          # nih-plug, tract-dsp, tiny-skia-widgets, criterion (dev)
  src/
    lib.rs            # Plugin, Params, process(), inline #[cfg(test)] tests
    editor.rs         # knob panel + retrig indicator + telemetry read
    hyper.rs          # voice bank: ring + N cubic-interp modulated taps + spread + width
    dimension.rs      # 4-tap widener: AM + Pitch modes, wet HPF, mono-safe sum
    transient.rs      # fast/slow envelope onset detector + refractory + telemetry write
    delay.rs          # modulated fractional delay line + 4-point cubic interpolation (NEW)
    lfo.rs            # phasor + waveforms (sine/tri), phase-in-turns (adapted pattern)
    main.rs           # standalone JACK binary
  benches/
    dsp.rs            # criterion benches
```

### Reuse vs. new
- **Reuse:** `imagine/midside.rs` (M/S for Dimension mono sum / side); `multosis` tremolo LFO pattern (phase-in-turns); `tiny-skia-widgets` (editor base, dial, selector, drag, text); `ctx.transport()` only if ever needed (not for v1).
- **New (the one real DSP piece):** the modulated fractional delay line with 4-point cubic interpolation (`delay.rs`).

## 11. Latency & real-time constraints

- **Reported latency: 0** (feed-forward dry + delayed wet; wet HPF is biquad/1-pole, no latency).
- No `Vec::new`, no `clone()` of collections, no `String` ops, no FFT in `process()`.
- All buffers pre-allocated in `initialize()`/`reset()`; sample-rate change handled there.
- `try_lock()` for any GUI→audio handoff (none needed beyond param system + atomics here).

## 12. Testing & benchmarks

### TDD (inline `#[cfg(test)]`)
- Unison=0 ⇒ Hyper passthrough equals dry.
- Dimension Mix=0 ⇒ passthrough equals input.
- **Dimension AM mode: `L_out + R_out` ≈ dry mono** (wet cancellation) within tolerance.
- Detune=0 ⇒ voices coherent (no detune; bounded gain only).
- Fractional delay: DC passes unchanged; a fixed fractional delay of a sine has expected magnitude/phase; never NaN/Inf.
- Transient detector: fires on a step/impulse; honors the refractory window (no double-fire); robust to input gain (ratio-based).
- Retrig zeros voice phases on fire.
- Output gain scales as expected; Bypass passes input.

### Benchmarks (`benches/dsp.rs`, per `nap/benches/dsp.rs`)
- `hyper/process` (sweep voice count 1/3/7), `dimension/process` (AM and Pitch), `transient/detect`, full `process` at 512-sample blocks @ 48 kHz.

## 13. References (research provenance)

- **Serum manual** (Xfer Records), Hyper/Dimension pp.27–29 — verbatim control descriptions; "micro-delay chorus 1–7 voices," "4 delay lines summed out-of-phase and slowly amplitude modulated," "Size," "Mix 0% disables."
- **Julius O. Smith / CCRMA (PASP):** Time-Varying Delay Effects, Doppler Simulation, Chorus Effect, Delay-Line Interpolation — Doppler detune from modulated delay; multi-tap chorus on one buffer; interpolation tradeoffs (cubic/Lagrange recommended under modulation).
- **Adam Szabo, "How to Emulate the Super Saw" (KTH, 2010):** measured JP-8000 asymmetric detune offsets, 11th-order nonlinear detune curve, center-vs-side gain laws.
- **Pseudo-stereo:** Lauridsen complementary-comb (`L=0.5(x−x_d)`, `R=0.5(x+x_d)`, `L+R=x` ⇒ mono-safe); Csound Journal pseudo-stereo survey; eContact review.
- **Roland Dimension-D / SDD-320:** Jürgen Haible circuit analysis (anti-phase LFO, opposite-polarity HPF cross-feed); Florian Anwander Roland-chorus tables (base ~7.5–10 ms, LFO 0.25–0.5 Hz, depth ±1.5–2.5 ms) — reference only; Serum diverges (AM not pitch).
