# miff Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `miff` — a convolution-filter plugin (VST3/CLAP/standalone) whose FIR kernel is hand-drawn with the MSEG editor instead of loaded from a wavetable.

**Architecture:** A new `miff/` crate modeled on `wavetable-filter`. The MSEG curve is baked into an FIR kernel (bipolar tap map + peak-magnitude normalization). Two convolution modes — Raw (direct SIMD time-domain) and Phaseless (fixed-frame STFT magnitude-only) — adapted from wavetable-filter's proven DSP into a clean `convolution.rs`. The GUI thread bakes the kernel and publishes it to the audio thread via a hand-rolled lock-free triple buffer. softbuffer + tiny-skia GUI embeds the MSEG editor widget in curve-only mode.

**Tech Stack:** Rust (nightly), nih-plug (fork), baseview + softbuffer + tiny-skia (CPU GUI), `tiny-skia-widgets` (incl. the `mseg` editor), `realfft`/`rustfft` (FFT). No new external dependency.

**Reference reading before starting:**
- `docs/superpowers/specs/2026-05-16-miff-design.md` — the design spec (authoritative).
- `wavetable-filter/` — the crate miff is modeled on. Especially `Cargo.toml`, `build.rs`, `src/main.rs`, `src/lib.rs` (the `FilterState`, the SIMD MAC loop, `process_stft_frame`), `src/editor.rs`, `src/editor/filter_response_view.rs`.
- `tiny-skia-widgets/src/mseg/mod.rs` — `MsegData`, `value_at_phase`, `warp`, the serde impl, `MAX_NODES`.
- `tiny-skia-widgets/src/mseg/{render,editor}.rs` — the MSEG editor: `draw_mseg`, `mseg_layout`, `MsegEditState` (`new_curve_only`, `on_mouse_down`/`_move`/`_up`, `on_double_click`, `on_right_click`, `set_stepped_draw`), `MsegEdit`.

**Test command:** `cargo nextest run -p miff` (whole crate) or `cargo nextest run -p miff <name>` for one test. Workspace lint: `cargo clippy -p miff -- -D warnings`.

**Key constants used throughout this plan:**
- `MAX_KERNEL: usize = 4096` — the maximum kernel length and the fixed Phaseless STFT frame size.
- `PHASELESS_HOP: usize = MAX_KERNEL / 2` = 2048 — the Phaseless overlap-add hop and its reported latency.

```rust
//! Curve → FIR-kernel bake, normalization, and the lock-free GUI→audio handoff.
//!
//! A self-contained module — no plugin or GUI types — so a future
//! workspace-wide shared-DSP refactor can lift it cleanly.

use tiny_skia_widgets::mseg::{warp, MsegData, MsegNode, MAX_NODES};

/// Maximum kernel length, and the fixed Phaseless STFT frame size.
pub const MAX_KERNEL: usize = 4096;
/// Real-FFT magnitude-spectrum bin count for a `MAX_KERNEL`-point transform.
pub const MAG_BINS: usize = MAX_KERNEL / 2 + 1;

/// A baked, normalized FIR kernel. Fixed-size and `Copy` so it crosses the
/// GUI→audio boundary with a lock-free copy that never allocates.
///
/// `taps[..len]` is the time-domain kernel (Raw mode). `mags` is the
/// magnitude spectrum of the zero-padded `MAX_KERNEL`-point kernel (Phaseless
/// mode and the response view). `is_zero` marks an all-zero kernel — miff
/// treats that as dry passthrough.
#[derive(Clone, Copy)]
pub struct Kernel {
    pub taps: [f32; MAX_KERNEL],
    pub len: usize,
    pub mags: [f32; MAG_BINS],
    pub is_zero: bool,
}

impl Default for Kernel {
    /// An all-zero kernel — dry passthrough.
    fn default() -> Self {
        Self {
            taps: [0.0; MAX_KERNEL],
            len: 256,
            mags: [0.0; MAG_BINS],
            is_zero: true,
        }
    }
}

/// miff's default document: a flat curve at value 0.5. Two nodes, both 0.5.
/// Bakes (bipolar map) to an all-zero kernel, i.e. clean dry passthrough — a
/// fresh miff colors nothing. NOT the MSEG core's `default()`, which is a ramp.
pub fn default_flat_curve() -> MsegData {
    let mut d = MsegData::default(); // ramp; we overwrite the two node values
    d.nodes[0] = MsegNode {
        time: 0.0,
        value: 0.5,
        tension: 0.0,
        stepped: false,
    };
    d.nodes[1] = MsegNode {
        time: 1.0,
        value: 0.5,
        tension: 0.0,
        stepped: false,
    };
    d.debug_assert_valid();
    d
}

/// Sample the curve at `phase` using a forward-only segment cursor `*seg`.
/// Callers walk `phase` monotonically increasing; `*seg` only advances. This
/// reproduces `tiny_skia_widgets::mseg::value_at_phase` exactly (stepped hold
/// + exponential `warp`), but without that function's rescan-from-node-0.
fn curve_value(data: &MsegData, phase: f32, seg: &mut usize) -> f32 {
    let a = data.active();
    let last = data.node_count - 1;
    // At or past the last node -> its value (matches value_at_phase).
    if phase >= a[last].time {
        return a[last].value;
    }
    // Advance the cursor to the last node (in 0..=last-1) with time <= phase.
    while *seg + 1 <= last - 1 && a[*seg + 1].time <= phase {
        *seg += 1;
    }
    let n0 = a[*seg];
    let n1 = a[*seg + 1];
    if n0.stepped {
        return n0.value;
    }
    let span = n1.time - n0.time;
    let t = if span > 1e-9 {
        (phase - n0.time) / span
    } else {
        0.0
    };
    n0.value + (n1.value - n0.value) * warp(t, n0.tension)
}

/// Bake the curve into raw (un-normalized) bipolar FIR taps, into
/// `out[..len]`. `kernel[i] = 2*curve_value(i/(len-1)) - 1` — the bipolar map
/// puts the MSEG midline (value 0.5) at a zero tap. `len` is clamped to
/// `[16, MAX_KERNEL]` and rounded down to a multiple of 16 (SIMD requirement).
/// Returns the effective `len` used.
pub fn bake_taps(data: &MsegData, len: usize, out: &mut [f32; MAX_KERNEL]) -> usize {
    let len = (len.clamp(16, MAX_KERNEL) / 16) * 16;
    *out = [0.0; MAX_KERNEL];
    let mut seg = 0usize;
    let denom = (len - 1).max(1) as f32;
    for (i, tap) in out.iter_mut().take(len).enumerate() {
        let phase = i as f32 / denom;
        *tap = 2.0 * curve_value(data, phase, &mut seg) - 1.0;
    }
    len
}
```

- [ ] **Step 3: Implement `bake` and the normalization**

Add to `kernel.rs` (module scope). The FFT uses `realfft`, the same crate wavetable-filter uses:

```rust
use realfft::RealFftPlanner;
use rustfft::num_complex::Complex;

/// Below this peak magnitude the kernel is treated as all-zero (dry
/// passthrough) — guards the normalization divide.
const ZERO_EPS: f32 = 1e-9;

/// Bake `data` into a normalized `Kernel` for a kernel of `len` taps.
///
/// Runs on the GUI thread (an O(`len` log `len`) FFT) — never the audio
/// thread. Steps: single-walk bake -> `MAX_KERNEL`-point FFT -> peak |H(k)|
/// -> divide taps and mags by the peak. A peak below `ZERO_EPS` (e.g. the
/// flat-0.5 default) yields an all-zero `is_zero` kernel.
pub fn bake(data: &MsegData, len: usize) -> Kernel {
    let mut taps = [0.0_f32; MAX_KERNEL];
    let len = bake_taps(data, len, &mut taps);

    // FFT the zero-padded MAX_KERNEL-point kernel for its magnitude spectrum.
    let mut planner = RealFftPlanner::<f32>::new();
    let fft = planner.plan_fft_forward(MAX_KERNEL);
    let mut fft_in = taps; // a MAX_KERNEL copy (taps beyond len are zero)
    let mut spectrum = vec![Complex::new(0.0_f32, 0.0); MAG_BINS];
    // `process` requires the input as a slice; the kernel is the padded copy.
    fft.process(&mut fft_in, &mut spectrum)
        .expect("FFT length matches planner");

    let mut mags = [0.0_f32; MAG_BINS];
    let mut peak = 0.0_f32;
    for (m, c) in mags.iter_mut().zip(spectrum.iter()) {
        *m = c.norm();
        peak = peak.max(*m);
    }

    if peak <= ZERO_EPS {
        return Kernel {
            taps: [0.0; MAX_KERNEL],
            len,
            mags: [0.0; MAG_BINS],
            is_zero: true,
        };
    }

    let inv = 1.0 / peak;
    for t in taps.iter_mut() {
        *t *= inv;
    }
    for m in mags.iter_mut() {
        *m *= inv;
    }
    Kernel {
        taps,
        len,
        mags,
        is_zero: false,
    }
}
```

```rust
use std::cell::UnsafeCell;
use std::sync::atomic::{AtomicUsize, Ordering};

/// Lock-free GUI→audio `Kernel` handoff. Three pre-allocated slots; the GUI
/// thread writes the slot that is neither `latest` nor the one a reader could
/// be holding, then publishes its index. The audio thread reads `latest`.
///
/// Single-producer (GUI) / single-consumer (audio). The producer never writes
/// the slot index the consumer last read, so a `read()` in progress is never
/// overwritten.
pub struct KernelTripleBuffer {
    slots: [UnsafeCell<Kernel>; 3],
    /// Index of the newest fully-written slot.
    latest: AtomicUsize,
    /// The slot the producer will write next (GUI-thread-only field).
    write_slot: UnsafeCell<usize>,
    /// The slot the consumer last read (audio-thread-only field).
    read_slot: UnsafeCell<usize>,
}

// SAFETY: access is disciplined SPSC — see the field docs. `publish` only ever
// touches a slot that is neither `latest` nor `read_slot`; `read` only reads
// `latest`. The atomics order the slot writes against the index publish.
unsafe impl Sync for KernelTripleBuffer {}

impl KernelTripleBuffer {
    /// A fresh buffer; every slot is the default (zero) kernel.
    pub fn new() -> Self {
        Self {
            slots: [
                UnsafeCell::new(Kernel::default()),
                UnsafeCell::new(Kernel::default()),
                UnsafeCell::new(Kernel::default()),
            ],
            latest: AtomicUsize::new(0),
            write_slot: UnsafeCell::new(1),
            read_slot: UnsafeCell::new(0),
        }
    }

    /// Publish a freshly-baked kernel. GUI thread only.
    pub fn publish(&self, kernel: Kernel) {
        // SAFETY: `write_slot` is GUI-thread-only; it is chosen (below) to be
        // neither `latest` nor a slot a reader holds, so this write races with
        // nothing.
        let w = unsafe { *self.write_slot.get() };
        unsafe { *self.slots[w].get() = kernel };
        // Release: the slot write above happens-before any Acquire load of
        // `latest` that observes `w`.
        self.latest.store(w, Ordering::Release);
        // Next write target: the slot that is neither the one just published
        // nor slot 0/1/2 currently — pick the remaining index. The consumer's
        // `read_slot` can only be the previously-published index, so cycling
        // 0->1->2->0 while skipping the just-published slot is safe.
        let next = (w + 1) % 3;
        // SAFETY: GUI-thread-only field.
        unsafe { *self.write_slot.get() = next };
    }

    /// Read the newest published kernel. Audio thread only. Returns a `Copy`
    /// of the slot — never blocks, never allocates.
    pub fn read(&self) -> Kernel {
        // Acquire: pairs with `publish`'s Release store.
        let l = self.latest.load(Ordering::Acquire);
        // SAFETY: `l` is a fully-written slot (the Release/Acquire pair orders
        // the slot write before this load observing `l`).
        let kernel = unsafe { *self.slots[l].get() };
        // SAFETY: audio-thread-only field — record what we read so a future
        // refinement could use it; harmless today.
        unsafe { *self.read_slot.get() = l };
        kernel
    }
}

impl Default for KernelTripleBuffer {
    fn default() -> Self {
        Self::new()
    }
}
```

```rust
//! Raw + Phaseless convolution engines, adapted from wavetable-filter's
//! proven DSP. A self-contained module — no plugin or GUI types.

use crate::kernel::{Kernel, MAX_KERNEL};
use std::simd::{num::SimdFloat, f32x16};

/// Per-channel time-domain convolution state: a double-buffered history ring
/// so `history_slice` can hand the MAC loop a contiguous newest-first window.
/// Adapted from wavetable-filter's `FilterState`.
pub struct RawChannel {
    history: Vec<f32>,
    write_pos: usize,
    mask: usize,
    is_silent: bool,
}

impl RawChannel {
    /// A channel sized for kernels up to `MAX_KERNEL` taps.
    pub fn new() -> Self {
        let cap = MAX_KERNEL.next_power_of_two();
        Self {
            history: vec![0.0; cap * 2],
            write_pos: 0,
            mask: cap - 1,
            is_silent: true,
        }
    }

    /// Zero the history.
    pub fn reset(&mut self) {
        self.history.iter_mut().for_each(|s| *s = 0.0);
        self.write_pos = 0;
        self.is_silent = true;
    }

    /// Push one input sample (double-buffered write).
    fn push(&mut self, sample: f32) {
        if sample.abs() > 1e-6 {
            self.is_silent = false;
        }
        let cap = self.mask + 1;
        self.history[self.write_pos] = sample;
        self.history[self.write_pos + cap] = sample;
        self.write_pos = (self.write_pos + 1) & self.mask;
    }

    /// Process one sample through `kernel` and return the filtered output.
    /// All-zero kernel -> the input is returned unchanged (dry passthrough).
    pub fn process(&mut self, sample: f32, kernel: &Kernel) -> f32 {
        self.push(sample);
        if kernel.is_zero {
            return sample; // dry passthrough — see the spec
        }
        if self.is_silent {
            return 0.0; // silence fast-path: history is all zero
        }
        let len = kernel.len;
        // Contiguous window of the `len` most recent input samples, from the
        // double-buffered history.
        let cap = self.mask + 1;
        let start = (self.write_pos + cap - len) & self.mask;
        let window = &self.history[start..start + len];
        // SIMD MAC. CORRECTNESS — the loop MUST compute the FIR convolution
        //   y[n] = sum_{k=0}^{len-1} taps[k] * x[n-k]
        // where `taps[k]` is the forward-order impulse response the bake
        // produced (`taps[0]` multiplies the current input x[n]). Whether the
        // kernel index is `c*16+j` or `len-1-(c*16+j)` depends on whether
        // `window` runs oldest-first or newest-first — that follows from how
        // `start`/the double-buffer are set up. Pick the pairing that makes
        // the asymmetric `known_kernel_yields_known_output` test pass; that
        // test is the oracle. The sketch below assumes `window` is
        // oldest-first (window[len-1] is the newest sample x[n]), so the
        // kernel is indexed reversed:
        let mut acc = f32x16::splat(0.0);
        let chunks = len / 16;
        for c in 0..chunks {
            let w = f32x16::from_slice(&window[c * 16..c * 16 + 16]);
            let mut kchunk = [0.0_f32; 16];
            for (j, kv) in kchunk.iter_mut().enumerate() {
                *kv = kernel.taps[len - 1 - (c * 16 + j)];
            }
            let k = f32x16::from_slice(&kchunk);
            acc += w * k;
        }
        acc.reduce_sum()
    }
}

impl Default for RawChannel {
    fn default() -> Self {
        Self::new()
    }
}
```

```rust
use crate::kernel::MAG_BINS;
use realfft::{ComplexToReal, RealToComplex};
use rustfft::num_complex::Complex;
use std::sync::Arc;

/// Phaseless STFT frame size (fixed — does not track `Length`).
pub const STFT_FRAME: usize = MAX_KERNEL;
/// Overlap-add hop (50% overlap) and the reported Phaseless latency.
pub const PHASELESS_HOP: usize = STFT_FRAME / 2;
/// Reported plugin latency in Phaseless mode, in samples.
pub const PHASELESS_LATENCY: u32 = PHASELESS_HOP as u32;

/// Per-channel STFT magnitude-only convolution state. Fixed `STFT_FRAME`-point
/// transform; a short kernel's magnitude spectrum is simply applied as-is
/// (the kernel was FFT'd zero-padded to `MAX_KERNEL`, so `Kernel::mags` is
/// already `STFT_FRAME`-resolution). Adapted from wavetable-filter's STFT path.
pub struct PhaselessChannel {
    fft: Arc<dyn RealToComplex<f32>>,
    ifft: Arc<dyn ComplexToReal<f32>>,
    window: Vec<f32>,           // STFT_FRAME Hann window
    in_buf: Vec<f32>,           // STFT_FRAME circular input
    in_pos: usize,
    out_buf: Vec<f32>,          // STFT_FRAME overlap-add accumulator
    out_pos: usize,
    scratch_time: Vec<f32>,     // STFT_FRAME
    scratch_freq: Vec<Complex<f32>>, // MAG_BINS
}

impl PhaselessChannel {
    pub fn new() -> Self {
        let mut planner = realfft::RealFftPlanner::<f32>::new();
        let fft = planner.plan_fft_forward(STFT_FRAME);
        let ifft = planner.plan_fft_inverse(STFT_FRAME);
        // Periodic Hann window.
        let window: Vec<f32> = (0..STFT_FRAME)
            .map(|n| {
                let x = std::f32::consts::PI * 2.0 * n as f32 / STFT_FRAME as f32;
                0.5 - 0.5 * x.cos()
            })
            .collect();
        Self {
            fft,
            ifft,
            window,
            in_buf: vec![0.0; STFT_FRAME],
            in_pos: 0,
            out_buf: vec![0.0; STFT_FRAME],
            out_pos: 0,
            scratch_time: vec![0.0; STFT_FRAME],
            scratch_freq: vec![Complex::new(0.0, 0.0); MAG_BINS],
        }
    }

    /// Zero all STFT state.
    pub fn reset(&mut self) {
        self.in_buf.iter_mut().for_each(|s| *s = 0.0);
        self.out_buf.iter_mut().for_each(|s| *s = 0.0);
        self.in_pos = 0;
        self.out_pos = 0;
    }

    /// Process one sample. The kernel's magnitude spectrum is applied to each
    /// STFT frame; phase is preserved. Output is delayed by `PHASELESS_HOP`.
    pub fn process(&mut self, sample: f32, kernel: &Kernel) -> f32 {
        // Write input into the circular buffer.
        self.in_buf[self.in_pos] = sample;
        self.in_pos = (self.in_pos + 1) % STFT_FRAME;

        // Read this sample's output from the overlap-add buffer, then clear it.
        let out = self.out_buf[self.out_pos];
        self.out_buf[self.out_pos] = 0.0;
        self.out_pos = (self.out_pos + 1) % STFT_FRAME;

        // Every PHASELESS_HOP samples, process a full frame.
        if self.out_pos % PHASELESS_HOP == 0 {
            self.process_frame(kernel);
        }
        out
    }

    /// FFT the latest windowed frame, apply the magnitude gains (preserving
    /// phase), IFFT, and overlap-add into `out_buf`.
    fn process_frame(&mut self, kernel: &Kernel) {
        // Extract the newest STFT_FRAME samples (newest at in_pos-1), windowed.
        for (i, slot) in self.scratch_time.iter_mut().enumerate() {
            let idx = (self.in_pos + i) % STFT_FRAME;
            *slot = self.in_buf[idx] * self.window[i];
        }
        self.fft
            .process(&mut self.scratch_time, &mut self.scratch_freq)
            .expect("STFT forward length matches planner");
        // Apply magnitude gains. A zero kernel -> gain 1.0 (passthrough).
        for (bin, c) in self.scratch_freq.iter_mut().enumerate() {
            let gain = if kernel.is_zero { 1.0 } else { kernel.mags[bin] };
            *c *= gain;
        }
        self.ifft
            .process(&mut self.scratch_freq, &mut self.scratch_time)
            .expect("STFT inverse length matches planner");
        // Overlap-add, scaled by 1/STFT_FRAME (realfft is unnormalized) and by
        // the window-overlap gain. Match wavetable-filter's exact scaling.
        let norm = 1.0 / STFT_FRAME as f32;
        for (i, &v) in self.scratch_time.iter().enumerate() {
            let idx = (self.out_pos + i) % STFT_FRAME;
            self.out_buf[idx] += v * norm;
        }
    }
}

impl Default for PhaselessChannel {
    fn default() -> Self {
        Self::new()
    }
}
```

```rust
    fn process(
        &mut self,
        buffer: &mut Buffer,
        _aux: &mut AuxiliaryBuffers,
        context: &mut impl ProcessContext<Self>,
    ) -> ProcessStatus {
        // Pick up the latest baked kernel (non-blocking; keep last on a miss).
        if let Some(k) = self.kernel_handoff.try_read() {
            self.kernel = k;
        }

        let mode = self.params.mode.value();
        // Click-safe mode switch: reset the path being switched INTO.
        if mode != self.last_mode {
            match mode {
                MiffMode::Raw => {
                    for ch in &mut self.raw {
                        ch.reset();
                    }
                }
                MiffMode::Phaseless => {
                    for ch in &mut self.phaseless {
                        ch.reset();
                    }
                }
            }
            self.last_mode = mode;
        }

        // Report latency only when it changes.
        let latency = match mode {
            MiffMode::Raw => 0,
            MiffMode::Phaseless => convolution::PHASELESS_LATENCY,
        };
        if latency != self.last_reported_latency {
            context.set_latency_samples(latency);
            self.last_reported_latency = latency;
        }

        for mut channel_samples in buffer.iter_samples() {
            let mix = self.params.mix.smoothed.next();
            let gain = self.params.gain.smoothed.next();
            for (ch, sample) in channel_samples.iter_mut().enumerate() {
                let ch = ch.min(1); // stereo state; mono uses channel 0
                let dry = *sample;
                let wet = match mode {
                    MiffMode::Raw => self.raw[ch].process(dry, &self.kernel),
                    MiffMode::Phaseless => self.phaseless[ch].process(dry, &self.kernel),
                };
                *sample = (dry + (wet - dry) * mix) * gain;
            }
        }

        ProcessStatus::Normal
    }
```

Verify `buffer.iter_samples()` / `channel_samples.iter_mut()` and the smoother `.smoothed.next()` API against wavetable-filter's `process()` — match its idioms exactly. `set_latency_samples` is on `ProcessContext` (or `InitContext`) — confirm which, against wavetable-filter, and call it correctly. If the buffer has more than 2 channels, the `ch.min(1)` keeps stereo state indices in range.

**Spec coverage** — the spec's requirements map to tasks:
- New `miff/` crate, Cargo.toml, main.rs, workspace member, xtask → Task 1.
- The curve→kernel bake (single-walk, bipolar map, distinct from `value_at_phase`) → Task 2.
- Peak-magnitude normalization → Task 3.
- The GUI→audio kernel handoff → Task 4 (resolved as `Arc<Mutex>` + `try_lock`, not a raw triple buffer — see the note below).
- Raw convolution (SIMD MAC, `len`-chunked, silence fast-path, all-zero → passthrough) → Task 5.
- Phaseless STFT (fixed 4096 frame, 2048 hop/latency) → Task 6.
- Params (Mode, Mix, Gain, non-automatable Length), persisted `MsegData`, persisted editor state → Tasks 1, 7, 9.
- `process()` (convolution dispatch, Mix, Gain, click-safe mode switch, latency-on-change) → Task 8.
- Softbuffer GUI: MSEG editor curve-only embed, control strip, event routing, re-bake → Tasks 9, 10, 11.
- Frequency-response view + input shadow → Task 12.
- Editor wired into the plugin, full integration → Task 13.
- Docs → Task 14.
- Testing (bake, normalization, convolution, plugin, render smoke) → distributed across Tasks 2-13.

**Deviation from the spec — the kernel handoff.** The spec describes a hand-rolled triple buffer ("three pre-allocated slots and an `AtomicUsize`"). A genuine triple buffer requires `unsafe` (`UnsafeCell` for the slot interior mutability), which the project's `CLAUDE.md` forbids except for FFI windowing glue. Task 4 therefore resolves the handoff as `Arc<Mutex<Kernel>>` + audio-thread `try_lock` — the exact pattern `wavetable-filter` already uses for its reload handoff, lock-free in practice (the kernel only changes on a GUI edit, so audio-thread contention is negligible) and `unsafe`-free. This satisfies the spec's intent (lock-free-in-practice GUI→audio `Kernel` handoff, no audio-thread allocation) while honoring the no-`unsafe` rule. Flagged here, not silently substituted.

**Adapted-DSP tasks (5, 6, 9, 10, 11, 12)** deliberately instruct "read wavetable-filter's X and reproduce it, adapting names/lengths" rather than reprinting hundreds of lines of wavetable-filter code that would drift from the source. The exact source locations are named. The *new* logic (the bake, the normalization, the handoff, the layout, the event-routing branch) is given in full. This is the honest decomposition for a crate explicitly specified as a "copied and adapted" near-twin.

**Type consistency** — `Kernel { taps: [f32; MAX_KERNEL], len, mags: [f32; MAG_BINS], is_zero }` is used identically in `kernel.rs`, `convolution.rs` (Raw reads `taps`, Phaseless reads `mags`), and `response_view.rs` (reads `mags`). `KernelHandoff::{new, publish, try_read}` is consistent across Tasks 4, 7, 8, 11. `bake(&MsegData, usize) -> Kernel` and `bake_taps(&MsegData, usize, &mut [f32; MAX_KERNEL]) -> usize` are consistent. `MiffMode::{Raw, Phaseless}`, `MiffParams`, `Miff` struct fields are consistent across Tasks 1, 7, 8, 13. The editor's `create(params, kernel_handoff, input_spectrum)` shape is consistent across Tasks 9, 12, 13.

**Note on `Length` and the bake.** `Length` is a non-automatable `IntParam`; it only changes via the GUI dial, so every re-bake is GUI-thread-triggered (Task 11's `rebake`, and Task 7's `initialize`). The audio thread never bakes — it only `try_read`s the published `Kernel`. This is the spec's load-bearing latency/allocation guarantee.
